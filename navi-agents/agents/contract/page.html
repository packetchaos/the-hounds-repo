Per-agent policy

Leave a policy blank to fall back to the risk-weighted top-N. Examples: IoT → "tag all IoT but not Dell or Intel — those are laptops"; Custom apps → "only /opt apps and jenkins matter"; Exposure routes → "payment routes → VM Team; docker → Custom Apps".

ACR changes

✨ Author ACR rules with AI — describe your criticality policy in plain English; the model maps it across your live tag list. Several rules in one sentence are fine (e.g. "drop all IoT and lab gear by 5, set anything internet-facing to 9, bump production databases to 10").
Change reason:
AI ACR authoring needs a language model. Set ANTHROPIC_API_KEY on the server to enable it — you can still add ACR rules by hand below.
