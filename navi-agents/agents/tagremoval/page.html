🗑️ Tag Removal

Garmr lists every tag in the tenant. Pick the ones to strip — each removal runs navi enrich tag … -remove against the assets that carry it and shows in the Tagging log as a remove job. Send your picks to the AI Contract and it will remove them first, pause 30 min, run navi update, then re-run the tagging workflow.

⚠ Removals depend on a current tags table. navi's -remove strips a tag by first clearing the assets navi.db believes carry it — that view is the local tags table, which is only rebuilt by an assets sync. If you tagged or changed assets and have not re-synced, a removal clears a stale (often empty) set and the live tag stays put. Run an assets update, let it finish, then remove.
